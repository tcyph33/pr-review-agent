# PR Review Agent

A TypeScript script that runs twice daily to find GitHub PRs requesting your review, runs each one through your `code-review` skill via Claude Code with full local repo access, tracks PR status (open, merged, closed), and saves structured feedback locally. A self-hosted HTML dashboard lets you read feedback, see review state, and open PRs or interactive Claude Code sessions from each card.

---

## How It Works

1. **launchd** triggers `run-reviews.ts` at 2am and 2pm via `tsx`
2. The script refreshes the status of all existing reviews (open, merged, closed)
3. It fetches your `code-review` skill live from your `claude-toolkit` repo
4. It resolves your GitHub username automatically from your token
5. For each PR, Claude Code runs headlessly via `claude --print`, invoking `/code-review <PR_URL>` with the skill as the system prompt — cloning the repo, reading full files, and grepping for issues
6. Results are saved to `results/reviews.json`
7. A macOS notification fires with a new/updated count (skipped on other platforms)
8. `server.ts` serves the dashboard at `http://localhost:3000/dashboard/`

## Setup

### 1. Clone and run setup

```bash
git clone https://github.com/tcyph33/pr-review-agent.git
cd pr-review-agent
bash setup-macos.sh
```

`setup-macos.sh` handles everything automatically:
- Installs npm dependencies
- Creates `logs/` and `results/` directories
- Copies `.env.example` to `.env` (if it doesn't already exist)
- Writes `~/.claude/settings.json` with pre-approved tools for autonomous review (safely merges if the file already exists)
- Generates both launchd plists with the correct absolute paths for this machine
- Loads both launchd jobs

### 2. Configure your keys

The script reads credentials from `process.env`. You have two options — pick either or both:

**Option A — `.env` file** (created by setup-macos.sh):
```
GITHUB_TOKEN=ghp_yourTokenHere
REVIEW_SKILL_URL=https://raw.githubusercontent.com/tcyph33/claude-toolkit/refs/heads/main/skills/code-review/SKILL.md
```

**Option B — environment variables** set in your shell profile or system:
```bash
export GITHUB_TOKEN=ghp_yourTokenHere
export REVIEW_SKILL_URL=https://raw.githubusercontent.com/tcyph33/claude-toolkit/refs/heads/main/skills/code-review/SKILL.md
```

If a variable is set in the environment it takes precedence over `.env`.

| Variable | Description |
|---|---|
| `GITHUB_TOKEN` | Personal access token. Needs `pull_requests: read` and `contents: read`. Settings → Developer settings → Fine-grained tokens. |
| `REVIEW_SKILL_URL` | Raw GitHub URL to your `code-review` skill `.md` file |

### 3. Authenticate GitHub CLI

```bash
gh auth login
```

### 4. Test manually

```bash
npm run review
```

## Dashboard

Open **http://localhost:3000/dashboard/** in any browser.

- Shows all reviewed PRs with agent feedback
- Summary bar: total PRs pending, repos, oldest PR age, average PR age
- Review state badge per card: Pending, Commented, Changes Requested, Approved
- PR status badge per card: Merged or Closed (open PRs shown first, merged/closed dimmed below)
- **Expand/Collapse** each feedback block
- **Open PR** opens that single PR in a new tab
- **Open All** opens every PR at once
- **⬡ Claude Code** opens an interactive Claude Code session in Terminal with full repo context and prior agent feedback loaded
- **Dismiss** removes a single review from the dashboard
- **Clear All** wipes all reviews after confirmation
- Auto-refreshes every 60 seconds

The dashboard server (`server.ts`) starts automatically at login via launchd and restarts itself if it ever crashes.

## Review Skill

The agent uses the `code-review` skill from your `claude-toolkit` repo. The skill:

- Clones the PR branch shallowly to `~/PR-Reviews/<repo>-<number>` using `gh`
- Reads full file contents — not just the diff — to catch issues in untouched dependent files
- Greps the entire repo for dangling references to removed or renamed concepts
- Runs a structured 9-category checklist (correctness, design, readability, completeness, semantic validity, testing, security, performance, documentation)
- Produces a structured report with a verdict: APPROVE, APPROVE WITH NITS, NEEDS CHANGES, or NEEDS DISCUSSION
- Does **not** post comments to GitHub — the review stays local

### Claude Code Permissions

`setup-macos.sh` automatically configures `~/.claude/settings.json` to pre-approve the exact tools the skill needs so reviews run without any manual approval prompts:

| Permission | Why |
|---|---|
| `read` | Reading files from the cloned repo |
| `bash(gh pr view*)` | Fetching PR metadata |
| `bash(gh pr diff*)` | Fetching the PR diff |
| `bash(gh repo clone*)` | Cloning the PR branch |
| `bash(mkdir*)` | Creating `~/PR-Reviews/` if it doesn't exist |
| `bash(grep*)` | Searching the full repo for dangling references |

No other tools are pre-approved. Claude Code will still prompt for anything outside this set.

### Open in Claude Code

Each dashboard card has an **⬡ Claude Code** button. Clicking it:

1. Fetches your review skill fresh from `REVIEW_SKILL_URL`
2. Opens a new Terminal window with Claude Code running interactively
3. Passes `/code-review <PR_URL>` as the first message along with the prior agent feedback as context
4. Claude Code runs the full skill against the local repo with the prior feedback already in view

This gives you an interactive follow-up session with the same context the agent had — full repo access, your skill, and the prior review.

## Scheduling (macOS only)

The review script runs at 2:00 AM and 2:00 PM daily via launchd (configured by `setup-macos.sh`). On macOS, a notification fires after each run showing how many reviews are new and how many were updated. On other platforms the notification is skipped silently.

> **Note:** launchd skips runs while the laptop is asleep and does not catch up on missed runs.
> If your laptop is usually closed at 2am, change the hour in the plist to a time it's reliably awake,
> or enable **System Settings → Energy → Wake for network access**.

### Useful commands

```bash
# Trigger the review script right now
launchctl start com.pr-review-agent

# View live logs
tail -f logs/out.log
tail -f logs/err.log

# Reload a job after editing its plist
launchctl unload ~/Library/LaunchAgents/com.pr-review-agent.plist
launchctl load ~/Library/LaunchAgents/com.pr-review-agent.plist

# Check both jobs are registered
launchctl list | grep pr-review
```

## Platform Notes

**Scheduling, the dashboard server, and the "Open in Claude Code" button are macOS only.** The `setup-macos.sh` script, launchd plists, and `launchctl` commands are all macOS-specific.

The `run-reviews.ts` script itself is fully cross-platform. If you want to run this on another OS, only the scheduling layer needs to change:
- **Linux** — use `cron`
- **Windows** — use Task Scheduler

### Running with the Laptop Lid Closed

By default, macOS puts the machine to sleep when the lid is closed, which causes launchd to skip scheduled runs. To allow the script to run with the lid closed:

1. Open **System Settings → Energy**
2. Enable **"Wake for network access"**

To change the schedule after setup:
```bash
open ~/Library/LaunchAgents/com.pr-review-agent.plist

launchctl unload ~/Library/LaunchAgents/com.pr-review-agent.plist
launchctl load ~/Library/LaunchAgents/com.pr-review-agent.plist
```
